# Step 1. PassMate MVP1 사용자 시나리오

## 목적

이 문서는 PassMate MVP1에서 사용자가 어떤 흐름으로 학습 자료를 만들고 복습하는지 정의한다.

구현 전 사용자의 실제 흐름을 먼저 이해하기 위한 기준 문서이다.

## 핵심 사용자

PassMate의 핵심 사용자는 시험, 자격증, 전공 과목, 업무 지식을 반복 학습해야 하는 개인 학습자이다.

사용자는 긴 자료를 직접 반복해서 읽기보다, 핵심 내용을 카드로 쪼개고 일정에 따라 복습하고 싶어 한다.

## MVP1 핵심 가치

- 학습 자료를 Deck과 Category로 정리한다.
- Material을 등록한다.
- Material에서 FlashCard를 만든다.
- FlashCard를 복습한다.
- 복습 결과를 ReviewHistory로 저장한다.

## 주요 시나리오

### 1. 학습 덱 만들기

사용자는 새로운 학습 주제를 만들기 위해 Deck을 생성한다.

예시:

- 정보처리기사
- 영어 단어
- Spring Boot 면접 준비

### 2. 카테고리 만들기

사용자는 Deck 안에서 학습 범위를 나누기 위해 Category를 만든다.

예시:

- 데이터베이스
- 네트워크
- 운영체제

### 3. 학습 자료 등록하기

사용자는 Category 안에 Material을 등록한다.

MVP1에서는 텍스트 기반 Material을 우선한다.

예시:

- 요약 문장
- 직접 입력한 개념 설명
- 강의나 책에서 정리한 내용

### 4. 플래시카드 만들기

사용자는 Material을 기반으로 FlashCard를 만든다.

MVP1에서는 사용자가 직접 카드의 앞면과 뒷면을 작성하는 흐름을 기본으로 한다.

### 5. 오늘의 복습 진행하기

사용자는 오늘 복습할 FlashCard 목록을 확인하고 카드 단위로 복습한다.

각 카드에 대해 사용자는 기억 여부 또는 난이도 결과를 남긴다.

### 6. 복습 기록 저장하기

복습 결과는 ReviewHistory로 저장된다.

이 기록은 이후 복습 일정과 학습 상태를 판단하는 근거가 된다.

## MVP1에서 제외되는 시나리오

아래 시나리오는 MVP1에서 구현하지 않는다.

- PDF 업로드 후 자동 카드 생성
- OCR
- AI 설명 생성
- AI 학습코치
- 퀴즈 모드
- 공유 Deck
- 결제

## MVP1 완료 기준

사용자가 다음 흐름을 끝까지 수행할 수 있으면 MVP1의 핵심 흐름이 완성된 것으로 본다.

```text
Deck 생성
  ↓
Category 생성
  ↓
Material 등록
  ↓
FlashCard 생성
  ↓
오늘의 복습
  ↓
ReviewHistory 저장
```
