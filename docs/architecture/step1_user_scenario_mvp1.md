# Step 1. PassMate MVP1 사용자 시나리오

## 목적

이 문서는 PassMate MVP1에서 사용자가 어떤 흐름으로 학습 세트를 만들고 복습하는지 정의한다.

구현 전 사용자의 실제 흐름을 먼저 이해하기 위한 기준 문서이다.

## 핵심 사용자

PassMate의 핵심 사용자는 시험, 자격증, 전공 과목, 업무 지식을 반복 학습해야 하는 개인 학습자이다.

사용자는 긴 자료를 직접 반복해서 읽기보다, 핵심 내용을 카드로 쪼개고 일정에 따라 복습하고 싶어 한다.

## UX 방향

PassMate MVP1의 카드 생성 경험은 퀴즐렛처럼 단순하게 만든다.

사용자는 먼저 학습 세트를 만들고, 같은 화면에서 카드의 앞면과 뒷면을 여러 줄로 빠르게 입력한다.

```text
폴더 선택 또는 생성
  ↓
학습 세트 생성
  ↓
카드 앞면 / 뒷면을 여러 개 입력
  ↓
저장
  ↓
복습 시작
```

## 사용자에게 보이는 용어

MVP1 화면에서는 아래 용어를 우선 사용한다.

```text
Deck      → 폴더
Material  → 학습 세트
FlashCard → 카드
Category  → 기본 섹션 또는 내부 분류
```

Category는 MVP1에서 사용자에게 필수 생성 단계로 노출하지 않는다. 필요한 경우 시스템이 기본 Category를 자동으로 사용하거나, 이후 고급 분류 기능으로 확장한다.

## 내부 도메인 매핑

기존 백엔드 도메인은 당장 크게 변경하지 않는다.

```text
Deck      = 폴더
Category  = 기본 섹션 또는 내부 분류
Material  = 학습 세트
FlashCard = 카드
```

즉, 사용자 경험은 `폴더 > 학습 세트 > 카드`로 단순화하지만 내부 데이터는 MVP1의 기존 구조를 활용한다.

## MVP1 핵심 가치

- 폴더로 학습 주제를 정리한다.
- 학습 세트를 만든다.
- 학습 세트 안에서 카드를 빠르게 여러 개 만든다.
- 오늘의 복습에서 카드를 한 장씩 복습한다.
- 복습 결과를 ReviewHistory로 저장한다.

## 주요 시나리오

### 1. 폴더 만들기

사용자는 학습 주제를 묶기 위해 폴더를 만든다.

예시:

- 정보처리기사
- 영어 단어
- Spring Boot 면접 준비

폴더는 사용자의 상위 정리 단위이다. 사용자는 폴더 안에 여러 학습 세트를 만들 수 있다.

### 2. 학습 세트 만들기

사용자는 폴더 안에 학습 세트를 만든다.

예시:

- 데이터베이스 정규화
- 네트워크 핵심 용어
- 운영체제 프로세스와 스레드

학습 세트는 사용자가 실제로 카드 묶음을 만들고 복습하는 기본 단위이다.

### 3. 카드 빠르게 만들기

사용자는 학습 세트 생성 또는 편집 화면에서 카드의 앞면과 뒷면을 여러 줄로 입력한다.

예시:

```text
앞면                         뒷면
제1정규형이란?               모든 속성 값이 원자값인 상태
트랜잭션 ACID란?             원자성, 일관성, 격리성, 지속성
HTTP 404란?                  요청한 리소스를 찾을 수 없음
```

카드 생성은 별도의 깊은 화면으로 이동하지 않고, 학습 세트 화면 안의 카드 입력 리스트, 모달, 또는 우측 패널에서 처리한다.

### 4. 학습 세트 상세 확인하기

사용자는 학습 세트 상세에서 다음을 확인한다.

- 학습 세트 제목
- 설명 또는 원본 메모
- 카드 목록
- 카드 추가/수정/삭제 진입
- 복습 시작 버튼

### 5. 오늘의 복습 진행하기

사용자는 오늘 복습할 카드 목록을 확인하고 카드 단위로 복습한다.

각 카드에 대해 사용자는 뒷면을 확인한 뒤 결과를 남긴다.

복습 결과:

- 다시
- 어려움
- 좋음
- 쉬움

### 6. 복습 기록 저장하기

복습 결과는 ReviewHistory로 저장된다.

이 기록은 이후 복습 일정과 학습 상태를 판단하는 근거가 된다.

## MVP1에서 제외되는 시나리오

아래 시나리오는 MVP1에서 구현하지 않는다.

- PDF 업로드 후 자동 카드 생성
- OCR
- AI 설명 생성
- AI 학습코치
- 퀴즈 모드
- 공유 Deck
- 결제

## MVP1 완료 기준

사용자가 다음 흐름을 끝까지 수행할 수 있으면 MVP1의 핵심 흐름이 완성된 것으로 본다.

```text
폴더 생성
  ↓
학습 세트 생성
  ↓
카드 여러 개 생성
  ↓
오늘의 복습
  ↓
ReviewHistory 저장
```
